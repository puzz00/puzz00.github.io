Windows Vulnerabilities Overview

These vulnerabilities exist in windows itself rather than third party services. Windows has native services and protocols such as smb. Below are some of the native services which are commonly found (and exploited) on windows.

IIS

Microsoft Internet Information Services is a web server which is developed by microsoft and runs on ports 80 or 443.

WebDAV

This is an http extension which turns a web server into a file server so clients can interact with files on the web server - they can move, copy, edit and delete files on the webserver using webdav. This service works with iis and runs on the same ports as it. This service tends to serve files rather than web apps.

SMB

smb runs on port 445 on newer windows versions but it used to run on netbios using port 139. This service enables files and devices to be shared in a windows based environment.

RDP

This runs on port 3389 and lets people remotely interact with machines via a gui - it is disabled by default but can be enabled - valid credentials will be needed to access it.

WinRM

This is the Windows Remote Management Protocol and it lets people execute commands on a windows box remotely via the internet or an internal network. It runs on port 5986 by default or 443 if it has an ssl certificate.
